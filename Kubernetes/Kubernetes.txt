Pod->The smallest unit of Kubernetes
   ‚û°Ô∏èA Pod is a portable wrapper around one or more containers.
   ‚û°Ô∏èIt offers shared networking and storage for containers.
   ‚û°Ô∏èWe can deploy multiple containers inside a single Pod.
   ‚û°Ô∏èContainers within a Pod can communicate easily since they share the same network.
   ‚û°Ô∏èPods are the smallest units in Kubernetes that run containers,A ReplicaSet manages the number of Pods running.

ReplicaSet‚û°Ô∏è‚û°Ô∏èIf one Pod crashes, the ReplicaSet will create a new one automatically to keep the total number the same.
            ‚û°Ô∏èIt ensures that the specified number of Pods are always active.
            ‚û°Ô∏èReplicaSet does not handle updates or rollbacks, which can lead to downtime during changes.

Deployment‚û°Ô∏è‚û°Ô∏èIt uses a rolling update strategy to replace Pods gradually without downtime.
            ‚û°Ô∏èTo upgrade from version v1 to v2 in Kubernetes, 
                 update the deployment image. Kubernetes will automatically replace the old Pods with new ones.
                 If version v1 was working fine and you want to go back to v1, the **ReplicaSet** doesn‚Äôt handle updates,
                 rollbacks. Rollbacks are managed by **Deployments** in Kubernetes.
                 ["Rolling updates replace pods without downtime."]

Service‚û°Ô∏è‚û°Ô∏èExposes the Pods to the network, allowing external or internal access.
